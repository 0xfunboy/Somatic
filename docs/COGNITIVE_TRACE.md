# COGNITIVE TRACE

## What it is

The Cognitive Trace is an **observable runtime trace** generated by the Soma
runtime from explicit state variables. It is NOT hidden LLM chain-of-thought.

Label in UI: **COGNITIVE TRACE**
Subtext: *Observable runtime trace — not hidden LLM chain-of-thought.*

## Phases

| Phase | Description |
|---|---|
| `perception` | Sensor read, provider quality, real/mock status |
| `body_model` | Derived stress, comfort, scenario classification |
| `somatic_projection` | Projector mode, norm, availability |
| `drives` | Dominant drive, all drive intensities |
| `goals` | Active goal, priority, progress |
| `policy` | Selected policy mode and reason |
| `action_selection` | Silent + visible actions chosen |
| `reflection` | Trigger, summary, learned facts |
| `memory_update` | Which memory fields were updated |
| `llm` | LLM status, mode, availability |
| `fallback` | Fallback/reflex mode active |
| `warning` | Any runtime anomaly |
| `growth` | Growth score/stage update |

## Schema

```json
{
  "timestamp": 1234567890.0,
  "phase": "perception",
  "summary": "Real Linux telemetry received. CPU 45°C, load 25%.",
  "inputs": {"provider": "linux", "is_real": true},
  "outputs": {"cpu_temp": 45.0, "source_quality": 0.8},
  "confidence": 0.8,
  "visible": true,
  "level": "info"
}
```

## Storage

Appended to: `data/mind/cognitive_trace.jsonl`
