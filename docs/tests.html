This page exposes the repo's useful smoke and validation suites through the live WebSocket runtime. Raw scripts remain in scripts/, but the operator sees only grouped checks with readable outcomes, step timing, and the exact failing command when something breaks.

The live backend is required for suites marked LIVE BACKEND. The operator console does not invent results: it renders the same structured events persisted by the runtime and the same curated validation commands the repo already uses.
